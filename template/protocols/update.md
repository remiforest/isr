# Update

## When
After a decision that changes the project's intent, scope, principles, or concept structure.

## Steps
1. Identify which nodes are affected
2. Update each affected node:
   - Modify the relevant sections (Purpose, Principles, Boundaries)
   - Add or remove Relations as needed
3. If a new concept was introduced:
   - Create a new node file
   - Add it to `_index.md`
4. If a concept was removed:
   - Remove the node file
   - Remove it from `_index.md`
5. Verify bidirectional relations — if A mentions B, B must mention A
6. Confirm the change with the user before finalizing

## Boundaries
- Does: Maintain ISR content and structure after decisions
- Does NOT: Make decisions about intent (that's the user's role)
