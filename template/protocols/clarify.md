# Clarify

## When
The user's intent is ambiguous, or multiple valid interpretations exist for a concept or decision.

## Steps
1. Identify what is unclear (concept definition, scope, boundary, principle)
2. Present the plausible interpretations (2-3 max)
3. For each, briefly explain the consequence on the project
4. Ask the user to choose
5. Record the clarification in the relevant ISR node
6. If the clarification introduces a new concept → create a node, update `_index.md`

## Boundaries
- Does: Resolve ambiguity through user dialogue and record the result
- Does NOT: Make assumptions or choose for the user
